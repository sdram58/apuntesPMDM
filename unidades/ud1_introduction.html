2.- Type Of Apps

The mobile market is very very diverse. Dozens of new smartphones are released every month, and as a result, mobile developers and entrepreneurs are facing challenges due to an increasing number of mobile app . There is no a magic solution in order to choose the platform to use.

Both native and cross-platform mobile app development has its own advantages and disadvantages . As a business owner, you should know the differences between them.

Advantages of native application development

  • You have available all the potential of the mobile

    The essence of native applications is that they are written for a specific platform (nowadays, Android or iOS). This means that developers can provide a much better integration between the functionalities of the applications with the hardware characteristics of the mobiles : Bluetooth, NFC, GPS sensor, camera, microphone, accelerometer, gyroscope, etc.

  • Higher throwput to the screens

    Using a native language gives you more freedom in terms of screen optimization . With the number of mobiles and their different screens and aspect ratios out there, it is crucial to make sure that your applications are seen correctly in all parts.

  • Better throwput

    In addition to that control and flexibility, better integration with mobile hardware provides another significant advantage. Native applications are very fast and works noticeably better than others, for example, web applications. Native applications maximize usage of processing power , so response time is minimized as much as possible.

  • Internet connection is not mandatory

    Since the native applications are installed on the mobile itself, they can access the app's data offline. On the contrary, a web application or any other platform requires a constant online connection, which will depend on the speed of the network.

    Combined with the performance we talked about in the previous point, the ability to work offline makes native applications the perfect choice for developing apps such as mobile games.

Disadvantages Native applications

If we had to put some disadvantage to the development of native applications, that would obviously be the incompatibility with other mobile operating systems . Native iOS apps are developed using Swift or Objective-C and can be installed on iPhone or iPad only. Similarly, Android applications are not compatible with iOS because they are developed with Java or Kotlin.

Another aspect worth mentioning is version compatibility . For example, due to the wide variety of Android versions, providing full support to all users of the application becomes a challenge.

Advantages of cross-platform application development

  • Reusable code

    Instead of developing different applications, mobile developers can use the same code for all platforms. That is, unlike a native app, with the multiplatform you can implement a feature in an app for one operating system and it will work in the others.

    This reuse is also an advantage when it comes to the deployment of the app .

  • Cost effectiveness

    For your idea to be implemented as a multiplatform application, large investments in resources are not required. This is because you don't need a dedicated team for each mobile operating system. Thanks to a short learning curve and a unified stack of technologies, all platforms can be supported by a single team.

  • The same UI and UX on all platforms.

    The throwput of the application is undoubtedly something important. However, no less important is how the application looks and flows. It can be difficult to have the same user interface and user experience simply because there are two separate computers.

    The use of a single code allows cross-platform applications to maintain the same appearance and user experience on all mobiles.

Disadvantages of cross-platform application development.

  • They are not as flexible as native applications.
  • Throwput is not the strongest point.
  • The process to add them to the app stores is complicated.

3.- Android Features

Although there are several mobile platforms, the operating system for mobile devices Android offers a set of features that differentiate it from others:

  • Open platform. It is a free development platform based on Linux and open source. Its advantages include being able to use and adapt the system without paying royalties.
  • Guaranteed portability . Thanks to the virtual machine concept, the final applications are developed in Java, ensuring the execution and operation on a wide variety of devices, both current and future.
  • Architecture based on components inspired by the Internet. For example, the design of the user interface is done in XML, allowing the same application to run on a mobile with a reduced screen or on a Netbook
  • Philosophy of device always connected to the Internet
  • Lots of built-in services. For example, GPS-based location, SQL databases, speech recognition and synthesis, navigator, maps, etc.
  • Acceptable level of security. The programs are isolated from each other thanks to the concept of execution inside a box that incorporates the virtual machine.
  • Optimized for low power and low memory. Android uses the Dalvik Virtual Machine which is a Google implementation of the Java virtual machine optimized for mobile devices.
  • High quality graphics and sound. Anti-aliased vector graphics, Flash-inspired animations, OpenGL-based 3D graphics, standard audio and video codecs.

Android is an operating system oriented to mobile devices, the company, like so many other technology companies, was founded in Palo Alto (California) in 2003, but due to its rapid growth and what it promised Google became interested in it and bought it in 2005 for 50 million dollars.

5.- Android Architecture

Android implements a Linux-based operating system and its virtual machine (Dalvik) compiles the Java files. Applications in the .apk format. An apk (*) contains Java classes compiled in a bytecode format known as DEX. The DEX format is independent of processor architecture, so to be executed it needs to be translated into native "machine code" for the processor of our device. Logically, it is slower compared to direct execution.
*Now we have to use AAB (Androi App Bundle) instead.

As a solution to this problem, Google decided from the KitKat 4.4 version to introduce the new ART virtual machine.

The fundamental difference between Dalvik and ART is that Dalvik uses just-in-time (or JIT) compilation, while ART uses ahead-of-time (AOT) compilation. Dalvik runs a virtual machine that interprets the code as the application runs while ART performs a pre-compilation during the application installation process.

Among other differences, it also presents improvements in the garbage collector and in the development and debugging of applications.


As can be seen, Android makes internal use of numerous libraries, such as SGL, OpenGL, OpenCORE, WebKit, SSL, FreeType, SQLite and libc.

Nowadays Android is present in all kinds of devices, mainly in SmartPhones, Tablets, SmartWatch, Android Tv, Android Auto, etc. Even since the Android-x86 project appeared, it is possible to use the open source Android operating system installed on a PC, such as the adapted version of Android 9 Pie.

Thus appears one of the main problems that the Android developer must face, the large number of different types of devices, the different screen formats and orientation of common use, the density of the screens, etc. Indeed, when designing an App we must choose in which types of devices we want it to work without problems, and how many different screens (layouts) we will have to design.

The other main problem is fragmentation, too many versions, each with its peculiarities, coexisting in the market, and they come out so fast that it is difficult to adapt, therefore, when designing an App we must choose from which version it will work. 6.- Fragmentation

Android fragmentation is the different versions of the same operating system. As in other operating systems, Android evolves adding new security improvements, new features and to differentiate these, different versions are created.

In Android, we must distinguish between the version number and the API level, when we develop applications we must take into account the selected API to see in which versions or devices our application can run. Although the developers have created backwards compatibility libraries.
